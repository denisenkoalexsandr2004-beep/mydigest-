# MyDigest Caution Source Review

Дата: 2026-06-06

Цель: повторно проверить все источники со статусом `Caution` и принять production-решение.

Статусы решения:

- `KEEP_RESTRICTED` — оставить, но только с низким весом, whitelist-доменом и сильными ключевыми словами.
- `QUERY_ONLY` — не использовать как общий домен; искать только через точный `site:domain + тема`.
- `MANUAL_BACKGROUND` — не использовать в автоматическом дайджесте, но можно держать как справочный источник для ручного исследования.
- `REMOVE_AUTO` — убрать из автоматического поиска MyDigest.

## Правило для backend

Для источников из `KEEP_RESTRICTED` и `QUERY_ONLY` нужно использовать отдельный повышенный порог:

- минимум 2 сильных совпадения с `requiredAny`;
- минимум 1 совпадение с department-specific logic;
- низкий source weight: `3-5`;
- не брать материал, если совпадение найдено только в названии источника, а не в заголовке/summary.
