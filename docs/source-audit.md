# MyDigest Source Audit

Дата аудита: 2026-06-06

Цель: проверить, подходят ли источники из `docs/source-catalog.md` под реальные задачи MyDigest.

Критерии:

- `Core` — источник точно подходит направлению, можно давать высокий вес.
- `Support` — источник полезен, но не должен доминировать в выдаче.
- `Caution` — источник достоверный, но высокий риск нерелевантных материалов; нужен строгий keyword/whitelist-фильтр.
- `Replace` — источник лучше заменить или не использовать в автоматической выдаче.

Важно: MyDigest должен искать не “любые новости”, а рабочие сигналы для отделов. Поэтому даже достоверный источник может получить статус `Caution` или `Replace`, если он слишком широкий, рекламный, редко обновляется или не дает нужный бизнес-контекст.

## Что нужно изменить перед production whitelist

1. Источники со статусом `Replace` уже удалены из каталога.
2. Источникам `Caution` поставить низкий вес и требовать сильное совпадение по направлению.
3. Для DDGS добавить whitelist доменов по каждому направлению.
4. Для широких медиа использовать не домен целиком, а связку `site:domain + query`.
5. Не давать корпоративным сайтам сетей доминировать: они полезны, но часто являются PR-повесткой.
6. Для выставок разделить источники на `индустрия выставок`, `сайты конкурирующих выставок`, `господдержка/экспорт`.
7. Для конференций ядро должно быть: Vademecum, Фармвестник, Remedium, Медвестник, GxP, Минздрав, Росздравнадзор, regulation.gov.ru.
8. Для ИИ ядро должно быть: OpenAI, Anthropic, DeepMind, Microsoft AI, Meta AI, NVIDIA, Hugging Face, TechCrunch AI, VentureBeat AI, The Decoder.
